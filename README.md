# Визуальная новелла (VN) для SillyTavern

Расширение превращает ответы AI в иммерсивную **визуальную новеллу**: полноэкранный плеер с фоном-сценой, диалоговым окном, навигацией по репликам и сценам. Картинки рисует уже установленное расширение **sillyimages** (Inline Image Generation) — это расширение их только раскладывает по кадрам.

Аналог популярного regex-скрипта «Visual Novel by 白桃», но как полноценное расширение, **на русском** и с интеграцией в **sillyimages** вместо st-chatu8.

## Как это работает

Есть два режима подачи картинок (переключатель **«Мини-ИИ сам ставит кадры»** во вкладке «Картинки»):

### Режим «Мини-ИИ сам ставит кадры» (по умолчанию ✓ — надёжное число картинок)

Основному ИИ **не уходит НИКАКОЙ инструкции** про картинки/формат/`<vn>`/`<vn-status>` — он просто отыгрывает как обычно. После ответа за дело берётся **мини-ИИ-режиссёр** (тот же «лёгкий ИИ» из настроек):

1. Читает готовый ответ основного ИИ.
2. Режет сцену **ровно на столько кадров, сколько задано** («от 2 до 3») и пишет к каждому промпт на английском.
3. Заодно одним вызовом заполняет статус сцены для всех панелей (место/настроение/время, отношения, эмоция, инвентарь, цели) и, если включена «Музыка», подбирает **плейлист под вайб** из N реальных песен; если задан каст — расставляет спрайты.
4. Вписывает теги картинок в сообщение и запускает их генерацию в твоём расширении картинок (sillyimages/megarakk).

> Плюс: число кадров соблюдается **всегда** (его ставит код, а не «если модель послушалась»), и основной промпт не засоряется служебной разметкой. Минус: на каждый ответ — один дополнительный вызов мини-ИИ (лучше подключить дешёвый/быстрый эндпоинт во вкладке «Основное» → «Мини-ИИ» и нажать **«Модели с сервера»**).

### Режим «теги в промпте» (выключи режиссёра)

1. Расширение инжектит в промпт инструкцию: AI сам оборачивает ответ в `<vn>…</vn>` и расставляет картинки тегами:
   ```html
   <img data-iig-instruction='{"prompt":"...","aspect_ratio":"16:9","image_size":"1K"}' src="[IMG:GEN]">
   ```
2. **sillyimages** ловит `[IMG:GEN]` и генерирует картинки.

В обоих режимах расширение в ленте прячет сырую разметку и показывает **карточку-лаунчер**; жмёшь «Открыть» → полноэкранный плеер.

> Картинки рисует твоё расширение картинок (sillyimages/megarakk), не sillyvn — это исключает конфликты. Режиссёр лишь готовит теги в нужном формате и запускает генерацию.

## Установка

Папка уже лежит в `SillyTavern/public/scripts/extensions/third-party/sillyvn/`. Просто перезапусти/обнови страницу SillyTavern.

Файлы:
```
sillyvn/
├── manifest.json
├── index.js
├── style.css
└── README.md
```

Нужно, чтобы рядом было установлено и **настроено** расширение `sillyimages` (Inline Image Generation) — с рабочим API-эндпоинтом для генерации картинок.

## Использование

1. Открой `Extensions → Визуальная новелла`, убедись что включено.
2. Проверь статус: панель скажет, найден ли `sillyimages`.
3. Пиши персонажу как обычно — AI начнёт отвечать в формате VN.
4. В ленте появится карточка **▶ Открыть** → запускается плеер.

## Режимы, переходы и эффекты

| Режим | Варианты |
|---|---|
| **Фон** | **Генерация** (картинки sillyimages) · **Фоны ST** (загруженные в таверну фоны) · **Тёмный** (без картинки — для текста/спрайта) |
| **Переход** | **Нет** · **Плавно** (кроссфейд) · **Кино** (кроссфейд + медленный зум Ken Burns) |
| **Спрайты (каст)** | несколько персонажей одновременно, каждый со своими позицией и эмоцией — кнопка 👥 (см. ниже) |
| **Печать текста** | реплика печатается по буквам (клик допечатывает) |
| **Скорость текста** | медленно · обычная · быстро · очень быстро |
| **Форматирование** | `*курсив*` (действия/мысли) и `**жирный**` (акцент) рендерятся как стиль, а не сырые символы |
| **Выделять прямую речь** | текст в кавычках (`«…»` / `"…"` / `“…”`) подсвечивается акцентным цветом — видно, где речь, а где повествование, без подписи имени |
| **Имя говорящего (по спрайту)** | плашка с именем активного `<sprite>` над репликой. По умолчанию **выкл**: имя берётся из последнего тега спрайта, а не из самого текста, поэтому для надёжного «кто говорит» лучше полагаться на выделение прямой речи. Включай, только если ведёшь сцену спрайтами-кастом и ИИ аккуратно ставит `<sprite>` перед каждой репликой |
| **Темп автоплея** | пауза между репликами в автоплее: медленно · обычно · быстро |
| **Кинорамка** | чёрные полосы сверху/снизу (киношный кадр) |
| **Виньетка** | мягкое затемнение краёв для читаемости |
| **Вид** | размер шрифта, ширина/высота окна, матовость стекла, «заполнять/целиком» |
| **Цвета** | полный кастом: **акцент интерфейса** (перекрашивает весь «синий» UI — кнопки, прогресс, стрелку, рамки), цвет текста, прямой речи, курсива, жирного, фон окна. Кнопка «Сбросить цвета». Применяется на лету и к плееру, и к самой панели настроек. ⚠️ Если включён «Тинт акцента под настроение», акцент в плеере подкрашивается под сцену поверх выбранного — выключи тумблер, чтобы держать строго свой цвет |

### Фоны из SillyTavern

В режиме **Фон → Фоны ST** жми **«Выбрать фон ST…»** — откроется сетка всех загруженных в таверну фонов (плюс «Текущий фон таверны»). Выбранный фон берётся ровно тем же URL, что использует сама ST. Так можно вести новеллу на своих готовых задниках, а спрайтами/генерацией добавлять персонажей.

### Спрайты-каст: несколько персонажей на сцене

Кнопка **👥** в плеере открывает панель **«Спрайты на сцене»**. Можно показать **несколько персонажей одновременно** (твой перс, партнёр, {{user}}, гости…), каждого — на своей позиции (край◀ · слева · центр · справа · край▶) и со своей эмоцией. Спрайты грузятся **прямо в расширении** — внешний Character Expressions для этого не нужен.

**Менеджер спрайтов** (кнопка «Менеджер спрайтов…» в панели каста или «Открыть менеджер спрайтов» в настройках):
- **Добавь персонажа** быстрыми кнопками **Персонаж / Игрок / NPC** (или «+ Персонаж»). У каждого есть **роль** (выпадашка в карточке): `персонаж` = {{char}} (главный герой), `игрок` = {{user}}, `NPC` = по имени. Для ролей «персонаж»/«игрок» сопоставление идёт по {{char}}/{{user}} автоматически — даже если ИИ пишет настоящее имя. Для NPC имя должно совпадать с тем, как ИИ зовёт его в тексте.
- Для каждого загрузи **картинки эмоций** (neutral / happy / sad / angry / surprised…). Картинки автоматически ужимаются до 1024px и хранятся локально в браузере (IndexedDB), не раздувая настройки.
- ⭐ помечает эмоцию **по умолчанию**; у персонажа можно задать **позицию по умолчанию**.

**Как двигаются спрайты:**
- **Авто-расстановка (ИИ)** — модель сама выводит и убирает персонажей и меняет им эмоции тегом `<sprite name="Имя" pos="left" emotion="happy">` (инструкция и список доступных персонажей/эмоций добавляются в системный промпт автоматически). Говорящий подсвечивается, остальные слегка притемняются («Фокус на говорящем»).
- **Ручной контроль** — в панели каста включаешь/выключаешь актёров, меняешь позицию и эмоцию вручную; это перекрывает ИИ. Кнопка **«Сбросить к авто»** возвращает управление модели.

Дополнительно остаётся **«живой» спрайт** из встроенного **Character Expressions** (строка «(текущий · Expressions)» в панели) — он отражает эмоцию последней реплики через `#expression-image`. Если используешь загруженный спрайт для того же персонажа, «живой» лучше выключить, чтобы не было дубля.

### Комбинируй

Например: **Фон = Фоны ST** + **Спрайт = Центр** + **Печать текста** + **Кинорамка** = чистая классическая новелла на твоих задниках. Или **Фон = Генерация** + **Переход = Кино** = живые AI-сцены с плавным зумом.

## На телефоне

Плеер адаптивный: `100dvh`, учёт «безопасных зон» (вырез/жесты), диалог на всю ширину экрана, панель кнопок прокручивается горизонтально пальцем, тап по экрану — следующая реплика. Карточка-лаунчер в ленте тоже компактнее на узком экране.

- **Горизонталь по кнопке 📱** — поворачивает весь интерфейс плеера на 90° (через CSS-трансформ), так что можно играть в ландшафте, держа телефон вертикально и не включая системный автоповорот. Поворот сохраняется при переходе между сценами; на выходе сбрасывается.
- **Прокрутка широкого фона** — если кадр шире экрана (горизонтальный кадр на узком/вертикальном экране), фон можно **протянуть пальцем влево-вправо**. Доступно только при «Картинка = Заполнять» (cover); в режиме «Целиком» (contain) кадр и так влезает целиком. Тап (без протяжки) по-прежнему листает реплику.

## Панели-модули и тег статуса `<vn-status>`

Панели **«Сцена»** (время · место · погода · настроение) и **«Любовь/отношения»** теперь берут данные из **компактного тега, который основной ИИ пишет прямо в ответе** — без отдельных медленных вызовов «лёгкого ИИ». Тег ставится в самом конце ответа внутри `<vn>` и **скрыт от прозы** в плеере:

```html
<vn-status>
place: тюрьма
mood: агрессивно, опасно
time: утро
weather: пасмурно
symp: -2
trust: 0
attr: +3
reason: твой холодный отказ раззадорил его хищные инстинкты
trait: непокорная
</vn-status>
```

- `symp/trust/attr` — это **изменения** (-10..10) за последний обмен репликами, как раньше считал лёгкий ИИ; они накапливаются в скрытые числа отношений.
- `music` (модуль «Музыка») — трек/настроение под сцену; расширка ищет его и проигрывает (см. ниже).
- `emotion` (модуль «Эмоция спрайта») — эмоция {{char}}; двигает его спрайт без отдельного `<sprite>`.
- `item` (модуль «Инвентарь») — `+роза` / `-ключ`: персонаж дал/забрал предмет.
- `goal` / `flag` (модуль «Дневник») — текущая цель и случившаяся веха.
- Инструкция собирается автоматически из включённых модулей и добавляется в системный промпт (вместе с инжектом VN-промпта). Поля появляются в `<vn-status>` только для **включённых** модулей.
- **Лёгкий ИИ — запасной вариант:** если в ответе тега `<vn-status>` (или нужных полей) нет, модуль доберёт данные старым способом через лёгкий ИИ. Если этот вызов **не удался** (кривой/недоступный отдельный эндпоинт, нет связи, превышено время ожидания) — панель показывает «не вышло… — ⟳ повторить» вместо вечного спиннера; клик по ⟳ повторяет запрос.
- **Внешний календарь:** блок `<datetime>…</datetime>` (date / time / weather) тоже **вырезается из прозы** и используется для часов сцены — точное время/дата/погода перебивают то, что мог написать ИИ. Если расширения-календаря нет, время даёт сам ИИ через `<vn-status>`.

## Музыка (модуль «BGM»)

Фоновая музыка, которая **реально играет** — треки тянутся из бесплатных музыкальных API (NetEase / QQ / Kugou через GDStudio / VKeys / Meting; те же источники, что у популярного «酒馆助手 Music Player»). Каждый отдаёт прямую ссылку-стрим, расширка проигрывает её в `<audio>`.

Включи модуль в `Extensions → Визуальная новелла → Движок модулей → Музыка (BGM)`. Кнопка 🎵 в плеере (или мини-кнопка в HUD-чипе) открывает лист:
- **Поиск** — найди трек по «артист песня», послушай (▶) или добавь (+) в плейлист.
- **Плейлист** — на каждый чат свой; клик играет, ✕ убирает. Запоминается.
- **Проигрыватель** — играть/пауза, ◀ ▶, перемотка по полосе, громкость, режим (повтор / случайно / один трек).

**Музыка по сцене:**
- В режиме **«Мини-ИИ сам ставит кадры»** мини-ИИ-режиссёр сам подбирает под вайб сцены **плейлист из N реальных песен** (N задаётся в `Модули → Музыка → Песен в плейлисте под вайб`, по умолчанию 5), находит их в муз-API и собирает плейлист. Если музыка уже играет — текущий трек не перебивается, обновляется только очередь (следующий пойдёт из нового плейлиста), чтобы не дёргать музыку на каждой реплике.
- В режиме «теги в промпте» основной ИИ кладёт в `<vn-status>` поле `music:` (например `lofi rain`), и при смене настроения расширка включает один подходящий трек.

Музыка продолжает играть, даже когда лист и сам плеер закрыты.

> Поиск/получение ссылки идут запросами к сторонним API (CORS). Если у тебя работает «酒馆助手 Music Player» — заработает и здесь (источники те же). Если API недоступны из твоей сети, поиск вернёт пусто.

## История реплик (бэклог)

Кнопка 🕘 в плеере открывает журнал всех реплик сцен текущего чата (как «log/backlog» в классических новеллах) — удобно перечитать, если всплывашка/реплика промелькнула. Клик по строке прыгает прямо к этой реплике в плеере.

## Галерея кадров

Кнопка 🖼 — сетка всех картинок, что сгенерировались по чату. Клик по кадру открывает плеер на той сцене и кадре.

## Ещё модули (через `<vn-status>`)

Включаются в `Движок модулей`, все `defaultOn:false`:

- **Эмоция спрайта из статуса** — ИИ пишет `emotion: angry`, и спрайт {{char}} меняет лицо без отдельного `<sprite>`. Нужны загруженные эмоции героя в Менеджере спрайтов (ключи — англ.: neutral/happy/sad/angry…). Явный `<sprite>` и ручной режим имеют приоритет.
- **Инвентарь / подарки** — `item: +роза, -ключ` (можно `+монета x3`). Полка-лоток в HUD; текущий инвентарь инжектится в промпт, чтобы персонаж помнил предметы и реагировал.
- **Дневник / цели** — `goal: …` (текущие задачи, заменяются) и `flag: …` (вехи, копятся). Цели — мини-список в HUD; и цели, и вехи инжектятся в промпт.
- **Итог сцены** — на последнем кадре карточка-сводка: Δсимпатии/доверия/влечения, новая черта, заработанные искры, подарки (из `<vn-status>` этого ответа).
- **Таймер на выборы** — ИИ может пометить напряжённую развилку `<choices time="8">`: над вариантами идёт убывающая полоса, по истечении сам жмётся первый доступный вариант.

## Свой промпт картинок (совместимость)

Если у тебя уже есть свой продуманный промпт для картинок (в карточке персонажа или пресете — например формата `<illust>` / `<image_lite>` с booru-тегами, стиль-кодами, правилами для демилюдей и т.п.), расширение умеет с ним **дружить**, а не навязывать свой:

- **Распознавать картинки в любой обёртке** (вкл по умолчанию) — плеер ловит теги `<img data-iig-instruction='…' src="[IMG:GEN]">` где угодно: внутри `<illust>`, `<image_lite>`, `<div>`, без `<vn>`. Текст-обёртки вырезаются, проза и картинки раскладываются по кадрам.
- **Инжектить наш VN-промпт** — выключи этот тумблер, если используешь свой промпт. Тогда правит твой промпт, а расширение просто рендерит результат в плеере. (Держать оба промпта включёнными не стоит — будет конфликт обёрток `<vn>` vs `<illust>`.)
- **16:9 поверх всего** — тумблер «Только горизонтальные кадры» теперь **жёстко** переписывает `aspect_ratio` в ответе на `16:9` ещё до генерации. Работает даже если твой промпт ставит `{{random:1:1::2:3::…}}` или вертикаль — на ПК всё равно будут широкие кадры.

Рекомендуемая связка для «своего промпта»: распознавание **ВКЛ**, наш инжект **ВЫКЛ**, горизонталь **ВКЛ**.

## Внешние блоки (ExtBlocks) для картинок

Тумблер **«Внешние блоки (ExtBlocks) для картинок»** (вкладка «Картинки») управляет тем, **где живут кадры**:

- **Включён** — картинки идут **не в текст поста**, а во внешние блоки сообщения (`message.extra.extblocks`). Пост остаётся чистой прозой (это и экономит контекст на следующих ходах, и не «выводит картинки в пост»). В режиме **«Мини-ИИ сам ставит кадры»** режиссёр кладёт кадры именно во внешние блоки, а в самом тексте оставляет невидимые плейсхолдеры — плеер сшивает их обратно по порядку. Картинки генерит твоё расширение картинок, читая их из блоков.
- **Требование:** в расширении картинок (sillyimages/megarakk) включи опцию **«Process external blocks»** — иначе оно не увидит кадры в блоках и не сгенерит их (расширение об этом предупредит тостом).
- Также этот режим **подхватывает** картинки из блоков, созданных **другими** расширениями (формат тега — sillyimages `<img data-iig-instruction>`).
- **Выключен** — кадры вставляются прямо в текст ответа (как раньше).

> Работает для формата тега **sillyimages** (его умеют читать/писать в блоках). Для формата `<image>` кадры остаются в тексте.

## Про стиль картинок

Глобальный стиль намеренно **не навязывается**. Системный промпт просит модель задавать **свой постоянный облик и стиль рисовки каждому персонажу** прямо в `prompt` картинки и держать их консистентными. Так у разных героев/сцен может быть разный визуальный язык.

## Заметки

- Формат картинок полностью совпадает с тем, что ждёт sillyimages, поэтому работает и автогенерация, и перегенерация одной картинки.
- Если кадр пустой — значит sillyimages ещё генерирует (показывается «кадр генерируется…»). Если генерация **упала** (sillyimages пометил кадр ошибкой) или **затянулась** (>30 с без результата — например API не настроен/молча сдох), вместо вечного спиннера появляется «кадр не сгенерировался — ⟳ Повторить»: клик перезапускает генерацию именно этого кадра.
- **Свайпы и накопительные эффекты.** Искры, отношения и инвентарь начисляются за ход **один раз**: повторный рендер, переоткрытие чата и свайп туда-обратно их не дублируют. При свайпе на **другой вариант последнего сообщения** эффекты прошлого варианта **откатываются**, и применяется новый — числа отношений остаются честными при любом числе пере-генераций. Откат точен для свайпа **последнего** сообщения; свайп более старого сообщения в середине чата всю цепочку дальше не пересчитывает.
- **Бюджет токенов / кэш промпта.** Системная инструкция VN разнесена на две части: тяжёлый **статичный гайд** (правила, пример `<vn>`, ростер спрайтов, формат `<vn-status>`) инжектится в **кэшируемую** позицию (`IN_PROMPT`, выше истории чата) — на провайдерах с кэшем промпта (Claude/OpenAI) он не пере-биллится каждый запрос, а берётся из кэша; меняется только при правке настроек/каста/модулей. На `depth=2` у точки генерации остаётся лишь **короткий реминдер формата** + **динамика** (текущий инвентарь и цели, которые меняются по ходу игры). Так контекст не раздувается, а адхеренс к формату сохраняется. Размер статичного гайда: ~600–800 ток (ядро) … ~1.3–1.6k (с каст-ростером и всеми модулями).
- **Стриминг.** Пока ответ печатается по токенам, при включённом «скрывать разметку» сырая разметка (`<vn>`, JSON картинок, поля `<vn-status>`) **не мелькает** в ленте: как только в потоке появляется `<vn>`/картинка, текст прячется и показывается карточка с пометкой «✍ печатается…». По завершении (или обрыве) генерации карточка сменяется обычной (число реплик/кадров). Накопительные эффекты (искры/отношения/инвентарь) применяются строго по завершении ответа, не во время печати.
- **Кривой/обрезанный ответ.** Если модель не закрыла служебный блок (`</vn-status>`, `</datetime>`, `</choices>`) — например ответ обрезали по лимиту токенов — поля статуса и варианты выбора **не протекают в прозу**: незакрытый блок вырезается до конца, а варианты из незакрытого `<choices>` всё равно показываются. Маркеры markdown-блоков кода (```` ``` ````), если модель их вставила вопреки запросу, тоже убираются из реплик.
- Отладка: `window.SVN_DEBUG = true` в консоли, плюс `window.sillyVN.open(messageId)`.
