# Визуальная новелла (VN) для SillyTavern

Расширение показывает ответы ИИ как **визуальную новеллу**: полноэкранный плеер с фоном-сценой, диалоговым окном и навигацией по репликам и сценам.

Сами картинки рисует твоё расширение генерации картинок — VN только раскладывает их по кадрам и красиво показывает.

## Как это работает

Картинки в кадры можно расставлять двумя способами (переключатель **«Мини-ИИ сам ставит кадры»** на вкладке «Картинки»).

### Мини-ИИ сам ставит кадры (по умолчанию)

Основной ИИ просто отыгрывает сцену как обычно — никаких инструкций про картинки и разметку он не получает. После ответа подключается **мини-ИИ-режиссёр** (лёгкий ИИ из настроек):

1. Читает готовый ответ.
2. Режет сцену **ровно на нужное число кадров** и пишет к каждому промпт.
3. Тем же вызовом заполняет статус сцены для всех панелей (место, настроение, отношения, эмоция и т.д.) и, если включена музыка, подбирает плейлист под вайб.
4. Вставляет теги картинок и запускает их генерацию.

Плюс: число кадров соблюдается **всегда**, а основной промпт не засоряется. Минус: на каждый ответ — один лишний вызов мини-ИИ (лучше подключить дешёвый/быстрый эндпоинт на вкладке «Основное» → «Мини-ИИ»).

### Теги в промпте

Расширение само добавляет в промпт инструкцию, и ИИ оборачивает ответ в `<vn>…</vn>` и расставляет картинки тегами. Удобно, если не хочешь лишний вызов мини-ИИ.

В обоих случаях в ленте видна аккуратная **карточка-лаунчер** вместо сырой разметки. Жмёшь «Открыть» → плеер.

## Установка

Папка лежит в `SillyTavern/public/scripts/extensions/third-party/sillyvn/`. Перезапусти или обнови страницу SillyTavern.

Нужно, чтобы было установлено и настроено расширение генерации картинок с рабочим API.

## Управление в плеере

| Действие | Как |
|---|---|
| Следующая реплика | клик по экрану · `→` · `Пробел` |
| Предыдущая реплика | `←` |
| Автоплей | кнопка ▶ / ⏸ |
| Перерисовать кадр | ⟳ |
| Сохранить кадр | ⬇ |
| Скрыть/показать диалог | 👁 |
| История реплик | 🕘 — журнал всех реплик, клик прыгает к нужной |
| Галерея кадров | 🖼 — сетка всех картинок чата |
| Музыка / плейлист | 🎵 |
| Спрайты на сцене | 👥 |
| Поворот в горизонталь (телефон) | 📱 |
| Предыдущая / следующая сцена | ⏮ ⏭ |
| Режимы и вид | кнопка-ползунки |
| Ответить, не выходя | поле ввода внизу + `Enter` |
| Выход | ✕ · `Esc` |

> Прокрутить широкий фон на узком экране — потяни картинку пальцем влево-вправо.

## Настройки

`Extensions → Визуальная новелла`:

- **Включить режим** — вкл/выкл.
- **Скрывать разметку** — в ленте видна только карточка-лаунчер.
- **Автооткрытие** — плеер сам открывается на новом VN-ответе.
- **Максимум картинок на ответ**.
- **Спрайты персонажей (каст)** — менеджер спрайтов + тумблер «ИИ сам расставляет спрайты».
- **Доп. указания модели** — свободный текст в системный промпт (жанр, тон и т.п.).

### Режимы и вид (кнопка-ползунки прямо в плеере, всё запоминается)

| Что | Варианты |
|---|---|
| **Фон** | генерация · загруженные фоны ST · тёмный (без картинки) |
| **Переход** | нет · плавно (кроссфейд) · кино (кроссфейд + медленный зум) |
| **Печать текста** | по буквам, регулируемая скорость |
| **Форматирование** | `*курсив*` и `**жирный**` рендерятся как стиль |
| **Прямая речь** | текст в кавычках подсвечивается акцентом |
| **Кинорамка / виньетка** | чёрные полосы · затемнение краёв |
| **Вид** | размер шрифта, ширина/высота окна, матовость, «заполнять/целиком» |
| **Цвета** | полный кастом: акцент интерфейса, текст, речь, курсив, жирный, фон окна |

## Спрайты-каст

Кнопка **👥** — несколько персонажей одновременно, каждый на своей позиции (край ◀ · слева · центр · справа · край ▶) и со своей эмоцией. Спрайты грузятся прямо в расширении.

**Менеджер спрайтов:** добавь персонажа кнопками **Персонаж / Игрок / NPC**, задай роль и загрузи картинки эмоций (neutral / happy / sad / angry / surprised…). Картинки ужимаются и хранятся локально в браузере. ⭐ — эмоция и позиция по умолчанию.

Два режима:
- **Авто (ИИ)** — модель сама выводит и убирает персонажей и меняет эмоции. Говорящий подсвечивается.
- **Вручную** — включаешь актёров и меняешь позы/эмоции сам; кнопка «Сбросить к авто» возвращает управление модели.

## Панели-модули и тег `<vn-status>`

Панели сцены, отношений и других модулей берут данные из компактного тега, который ИИ пишет в конце ответа. Тег **скрыт от прозы** в плеере:

```
<vn-status>
place: тюрьма
mood: агрессивно, опасно
time: утро
symp: -2
trust: 0
attr: +3
trait: непокорная
</vn-status>
```

- `symp / trust / attr` — изменения (-10..10) за последний обмен; копятся в скрытые числа отношений.
- Поля появляются только для **включённых** модулей; инструкция собирается автоматически.
- Если тега в ответе нет, данные доберёт лёгкий ИИ. Если и это не вышло — панель покажет «⟳ повторить» вместо вечного спиннера.

### Модули (включаются в «Движок модулей»)

- **Сцена** — время · место · погода · настроение.
- **Любовь/отношения** — симпатия, доверие, влечение.
- **Музыка (BGM)** — фоновая музыка, которая реально играет (см. ниже).
- **Эмоция спрайта** — `emotion: angry` двигает спрайт без отдельного тега.
- **Инвентарь / подарки** — `item: +роза, -ключ`; полка в HUD, текущий инвентарь помнится в промпте.
- **Дневник / цели** — `goal:` (текущие задачи) и `flag:` (вехи); оба попадают в промпт.
- **Итог сцены** — на последнем кадре карточка-сводка: Δ отношений, новая черта, подарки.
- **Таймер на выборы** — напряжённую развилку можно пометить `<choices time="8">`: над вариантами убывающая полоса.

## Музыка

Фоновая музыка тянется из бесплатных музыкальных API и **реально играет**. Кнопка 🎵 в плеере открывает лист:

- **Поиск** — найди трек по «артист песня», послушай или добавь в плейлист.
- **Плейлист** — на каждый чат свой, запоминается.
- **Проигрыватель** — играть/пауза, перемотка, громкость, повтор/случайно/один трек.

В режиме «Мини-ИИ сам ставит кадры» режиссёр сам подбирает плейлист под вайб сцены. Если музыка уже играет, текущий трек не перебивается — обновляется только очередь. Музыка продолжает играть, даже когда лист и плеер закрыты.

> Поиск идёт запросами к сторонним API. Если они недоступны из твоей сети, поиск вернёт пусто.

## Свой промпт картинок

Если у тебя уже есть свой промпт для картинок, расширение умеет с ним дружить:

- **Распознавать картинки в любой обёртке** (вкл по умолчанию) — плеер ловит теги картинок где угодно, даже без `<vn>`.
- **Инжектить наш VN-промпт** — выключи, если используешь свой промпт. Тогда правит твой промпт, а расширение просто рендерит результат.
- **Только горизонтальные кадры** — жёстко переписывает соотношение сторон на `16:9` ещё до генерации.

Рекомендация для своего промпта: распознавание **ВКЛ**, наш инжект **ВЫКЛ**, горизонталь **ВКЛ**.

## Внешние блоки для картинок

Тумблер **«Внешние блоки для картинок»** управляет тем, где живут кадры:

- **Включён** — картинки идут не в текст поста, а во внешние блоки сообщения. Пост остаётся чистой прозой и экономит контекст. Нужно, чтобы в расширении картинок была включена обработка внешних блоков.
- **Выключен** — кадры вставляются прямо в текст (как раньше).

## Про стиль картинок

Глобальный стиль намеренно **не навязывается**. Системный промпт просит модель задать каждому персонажу свой постоянный облик и держать его консистентным.

## Заметки

- Если кадр пустой — картинка ещё генерируется. Если генерация упала или затянулась (>30 с), вместо вечного спиннера появится «⟳ Повторить».
- **Свайпы.** Искры, отношения и инвентарь начисляются за ход один раз — повторный рендер и свайп туда-обратно их не дублируют. При свайпе на другой вариант последнего сообщения эффекты прошлого варианта откатываются.
- **Стриминг.** Пока ответ печатается, сырая разметка не мелькает в ленте — показывается карточка «✍ печатается…». Эффекты применяются строго по завершении ответа.
- **Бюджет токенов.** Тяжёлый статичный гайд инжектится в кэшируемую позицию (не пере-биллится каждый запрос), а у точки генерации остаётся лишь короткий реминдер + текущая динамика.
- Отладка: `window.SVN_DEBUG = true` в консоли, плюс `window.sillyVN.open(messageId)`.
